## Auth Stack Decision: Better Auth replaces Clerk (19:00 UTC)

**Decision:** Replace Clerk with Better Auth across all projects.

**Why:**
- Open source, self-hosted, $0/mo at any scale
- Clerk charges per-MAU ($0.02/MAU after 10K) + add-ons ($100/mo for MFA, $100/mo for admin, $75/mo per SSO connection)
- At scale, Clerk = $500+/mo easily. Better Auth = $0 (just infra costs we already pay)
- Self-hosted = data stays on our Railway infra, no third-party dependency
- TypeScript-native, framework-agnostic (Next.js, Remix, Hono, etc.)
- Built-in: email/password, OAuth (Google/GitHub/Apple), magic links, passkeys, 2FA, email OTP
- Plugin ecosystem: multi-tenancy, organizations, SSO, payment integrations
- Works with PostgreSQL (already have), Prisma/Drizzle adapters
- AI tooling integration (LLMs.txt, MCP server)

**Impact:**
- Updated GTM infrastructure plan (`/docs/gtm-infrastructure-plan.md`)
- Cookie-cutter stack now: Next.js 15 → Better Auth → Stripe → PostHog → Blotato → Remotion → Railway
- FlipMyEra needs migration from Clerk → Better Auth (if Clerk is still in there)
- All new projects use Better Auth from day one

**Migration pattern:**
```
npm install better-auth
# Create auth.ts with betterAuth() config
# Point at existing Railway PostgreSQL
# CLI generates schema: npx better-auth generate
# Migrate routes from @clerk/nextjs to better-auth/react
```
